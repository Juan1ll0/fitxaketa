# Plan de Implementación: 003.2 - Tab Fichar

## Resumen

Implementar la pantalla completa del tab Fichar (ruta `/`) con: fecha actual formateada, cronómetro grande reactivo, indicador de estado (Trabajando/Descansando), botón Start/Stop con feedback visual, y resumen del día (total horas + número de jornadas). **Usa el store global centralizado** (`app-state.ts`) para obtener jornadas y resumen del día, evitando cargar datos localmente.

## Estado de la Spec

**Status:** draft  
**Dependencias:** 003.1-navegacion.md (Bottom Nav debe estar implementado)

## Análisis de Criterios de Aceptación

### AC1: Fecha actual

- Visible en parte superior
- Formato largo español: "Lunes, 21 de junio de 2026"
- Actualización automática al cambiar de día
- Función utilitaria `formatearFecha(date: Date): string`

### AC2: Cronómetro

- Grande y centrado (text-6xl o similar)
- Formato HH:MM:SS
- Actualización reactiva cada segundo
- Reutiliza `getElapsed()` del store

### AC3: Indicador de estado

- Texto: "Trabajando" (color primario) / "Descansando" (color secundario)
- Posición: debajo del cronómetro
- Icono opcional: punto parpadeante

### AC4: Botón Start/Stop

- Grande y accesible
- Texto reactivo: "Fichar entrada" / "Fichar salida"
- Color reactivo: primario (entrada) / danger (salida)
- Llama a `startJornada()` / `stopJornada()` del store
- Feedback visual (hover, active)

### AC5: Resumen del día

- Total horas fichadas hoy (jornadas cerradas + tiempo transcurrido si hay abierta)
- Número de jornadas hoy
- Formato: "Hoy: 7h 30m | 2 jornadas"
- Actualización reactiva
- Función utilitaria `calcularResumenDia(jornadas: Jornada[])`

### AC6: Persistencia

- Cronómetro persiste ante recargas (basado en `start_time` de jornada abierta)
- `export const ssr = false`

```typescript
export function formatearFecha(date: Date): string {
	return new Intl.DateTimeFormat('es-ES', {
		weekday: 'long',
		year: 'numeric',
		month: 'long',
		day: 'numeric'
	}).format(date);
}
```

4. **Implementar `calcularResumenDia`** — Función pura que calcula total de horas y número de jornadas:
   ```typescript
   export function calcularResumenDia(jornadas: Jornada[]): ResumenDia {
   	const totalMinutos = jornadas.reduce((acc, j) => acc + (j.duration || 0), 0);
   	return {
   		totalHoras: totalMinutos / 60,
   		totalJornadas: jornadas.length
   	};
   }
   ```
   **Nota:** Esta función solo suma jornadas cerradas (`duration` no null). La jornada abierta se maneja en el componente.

```svelte
<script lang="ts">
	import { onMount, onDestroy } from 'svelte';
	import {
		initAppState,
		startJornada,
		stopJornada,
		getClockedIn,
		getElapsed,
		subscribe
	} from '$lib/stores/app-state';
	import { getAllJornadas, type Jornada } from '$lib/db';
	import { formatearFecha, calcularResumenDia } from '$lib/utils/dashboard';

	export const ssr = false;

	let clockedIn = $state(false);
	let elapsed = $state('00:00:00');
	let hoy = $state('');
	let resumenDia = $state({ totalHoras: 0, totalJornadas: 0 });
	let jornadas = $state<Jornada[]>([]);

	let unsubscribe: (() => void) | null = null;
	let fechaInterval: ReturnType<typeof setInterval> | null = null;

	onMount(async () => {
		await initAppState();
		hoy = formatearFecha(new Date());

		// Suscribirse a cambios del store
		unsubscribe = subscribe(() => {
			clockedIn = getClockedIn();
			elapsed = getElapsed();
		});

		// Verificar cambio de día cada minuto
		fechaInterval = setInterval(() => {
			const nuevaFecha = formatearFecha(new Date());
			if (nuevaFecha !== hoy) {
				hoy = nuevaFecha;
				cargarJornadasHoy();
			}
		}, 60000);

		await cargarJornadasHoy();
	});

	onDestroy(() => {
		if (unsubscribe) unsubscribe();
		if (fechaInterval) clearInterval(fechaInterval);
	});

	async function cargarJornadasHoy() {
		const todas = await getAllJornadas();
		const inicioHoy = new Date();
		inicioHoy.setHours(0, 0, 0, 0);

		jornadas = todas.filter((j) => j.start_time >= inicioHoy);

		// Calcular resumen incluyendo jornada abierta
		const resumen = calcularResumenDia(jornadas.filter((j) => j.status === 'closed'));

		// Si hay jornada abierta, sumar tiempo transcurrido
		if (clockedIn) {
			const [h, m, s] = elapsed.split(':').map(Number);
			const minutosTranscurridos = h * 60 + m + s / 60;
			resumen.totalHoras += minutosTranscurridos / 60;
			resumen.totalJornadas += 1; // Contar la jornada abierta
		}

		resumenDia = resumen;
	}

	async function handleFichar() {
		if (clockedIn) {
			await stopJornada();
		} else {
			await startJornada();
		}
		await cargarJornadasHoy();
	}
</script>
```

## Riesgos Técnicos

### R1: Memory Leak por Suscripción

**Descripción:** La suscripción al store `app-state.ts` puede causar memory leaks si no se limpia al destruir el componente.  
**Mitigación:** Guardar el `unsubscribe` retornado por `subscribe()` y llamarlo en `onDestroy`.

### R2: Cálculo Incorrecto del Resumen

**Descripción:** El resumen del día debe incluir tanto jornadas cerradas como el tiempo transcurrido de la jornada abierta actual.  
**Mitigación:** Filtrar jornadas cerradas para `calcularResumenDia`, luego sumar tiempo transcurrido de jornada abierta manualmente en el componente.

### R3: Fecha No Se Actualiza

**Descripción:** Si la app permanece abierta al cambiar de día, la fecha mostrada puede quedar desactualizada.  
**Mitigación:** Usar `setInterval` cada 60 segundos para verificar cambio de día y actualizar. Limpiar en `onDestroy`.

### R4: SSR con Dexie

**Descripción:** Dexie/IndexedDB no existe en servidor, causando errores de build si SSR está habilitado.  
**Mitigación:** Asegurar `export const ssr = false` en `+page.svelte`.

### R5: Token de Color Faltante

**Descripción:** Si `--color-danger` no está definido en `src/app.css`, el botón "Fichar entrada" no tendrá estilo correcto.  
**Mitigación:** Verificar que todos los tokens necesarios existan en `src/app.css` antes de implementar.

### R6: Dependencia de 003.1

**Descripción:** Esta sub-spec depende de que el Bottom Nav de 003.1 esté implementado.  
**Mitigación:** Verificar que 003.1 esté completa antes de iniciar implementación. Si no lo está, implementar de forma aislada y probar sin Bottom Nav.
