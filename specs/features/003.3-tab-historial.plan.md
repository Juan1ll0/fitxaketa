# Plan de Implementación: 003.3 Tab Historial

## Resumen

Implementar la vista de historial con jornadas agrupadas por día, migrando la funcionalidad existente de `/registros` a `/historial`. Incluye utilidades de formateo, componentes reutilizables y estado vacío.

```typescript
export function agruparPorDia(jornadas: Jornada[]): Map<string, Jornada[]> {
  const grupos = new Map<string, Jornada[]>();
  const hoy = new Date();
  hoy.setHours(0, 0, 0, 0);
  const ayer = new Date(hoy);
  ayer.setDate(ayer.getDate() - 1);

  for (const j of jornadas) {
    const fecha = new Date(j.start_time);
    fecha.setHours(0, 0, 0, 0);

    let key: string;
    if (fecha.getTime() === hoy.getTime()) {
      key = 'Hoy';
    } else if (fecha.getTime() === ayer.getTime()) {
      key = 'Ayer';
    } else {
      key = new Intl.DateTimeFormat('es-ES', {
        day: '2-digit',
        month: 'short',
        year: 'numeric'
      }).format(fecha);
    }

    if (!grupos.has(key)) grupos.set(key, []);
    grupos.get(key)!.push(j);
  }

  return grupos;
}

export function formatearHora(date: Date): string {
  return new Intl.DateTimeFormat('es-ES', {
    hour: '2-digit',
    minute: '2-digit'
  }).format(date);
}

export function formatearDuracion(minutos: number | null): string {
  if (minutos === null) return 'En curso';
  const h = Math.floor(minutos / 60);
  const m = minutos % 60;
  return `${String(h).padStart(2, '0')}:${String(m).padStart(2, '0')}:00`;
}
```

- Sin dependencias de UI

## Riesgos Técnicos

### 1. Tipo de `jornada.start_time` y `jornada.end_time`

**Riesgo:** La spec asume que son `Date` objects, pero Dexie podría almacenarlos como strings ISO.

**Mitigación:**
- Verificar en `src/lib/db.ts` el tipo exacto de `Jornada.start_time` y `Jornada.end_time`
- Si son strings, ajustar `formatearHora()` para aceptar `string | Date` y convertir internamente
- Añadir conversión en `agruparPorDia()` si es necesario

**Acción requerida:** `@impl-svelte` debe verificar el tipo antes de implementar.

### 2. Unidad de `jornada.duration`

**Riesgo:** La spec asume minutos, pero podría ser segundos.

**Mitigación:**
- Verificar en `src/lib/db.ts` la unidad de `Jornada.duration`
- Si es segundos, ajustar `formatearDuracion()`:
  ```typescript
  export function formatearDuracion(segundos: number | null): string {
    if (segundos === null) return 'En curso';
    const h = Math.floor(segundos / 3600);
    const m = Math.floor((segundos % 3600) / 60);
    const s = segundos % 60;
    return `${String(h).padStart(2, '0')}:${String(m).padStart(2, '0')}:${String(s).padStart(2, '0')}`;
  }
  ```

**Acción requerida:** `@impl-svelte` debe verificar la unidad antes de implementar.

### 3. Token `bg-surface-secondary` y `bg-background`

**Riesgo:** Estos tokens podrían no existir en `src/app.css`.

**Mitigación:**
- Verificar tokens disponibles en `src/app.css` dentro de `@theme {}`
- Si no existen, usar tokens alternativos:
  - `bg-surface-secondary` → `bg-surface` o `bg-gray-800`
  - `bg-background` → `bg-surface` o eliminar (usar `min-h-screen` sin fondo)

**Acción requerida:** `@impl-svelte` debe verificar tokens antes de implementar.

### 4. Orden de grupos en `Map`

**Riesgo:** `Map` mantiene orden de inserción, pero `agruparPorDia()` no garantiza orden descendente.

**Mitigación:**
- Ordenar jornadas antes de agrupar:
  ```typescript
  const jornadasOrdenadas = [...jornadas].sort(
    (a, b) => new Date(b.start_time).getTime() - new Date(a.start_time).getTime()
  );
  ```
- O ordenar el `Map` después:
  ```typescript
  const gruposOrdenados = new Map(
    [...grupos.entries()].sort((a, b) => {
      // Lógica de ordenamiento por fecha
    })
  );
  ```

**Acción requerida:** `@impl-svelte` debe implementar ordenamiento descendente.

### 5. Migración de tests

**Riesgo:** Los tests de `/registros` podrían ser muy específicos de la estructura DOM anterior.

**Mitigación:**
- Reescribir tests desde cero basándose en los ACs de esta spec
- No intentar migrar tests línea por línea

**Acción requerida:** `@tester` debe reescribir tests, no migrar ciegamente.
